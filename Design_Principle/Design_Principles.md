** There are Few things to be in our System **

1. Scalability :- Adding new features is Straight Forward.

2. Maintainabilty:- Adding new component will have less impact on other components.

3. Testability:- Decoupled design makes unit test easier.

4. Readability:- Clear seperation of concerns.


** Design Principles **

SOLID principles

1. Single Responsibility:- Each class will have one responsibility.

2. Open/Closed:- Open for Extension and Closed for Modification

3. Liskov Substitution Principle:- Child Class should substitute parent class seamlessly.

4. Interface Segregation Principle:- Invidual Interface for each class.

5. Dependency Inversion:- Making loosely coupled so testability increases so that each class can be tested easily just by pasing the arguments.

6. DRY:- Don't Repeat Yourself can use one logger clas instead of creating the class multiple times.

7. KISS:- Keep It Simple and Stupid

8. YAGNI:- You are not gonna need it with out adding unnecessary features.
